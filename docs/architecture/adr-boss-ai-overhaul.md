# ADR: Boss AI Overhaul + Voice/Personality System

**Date:** 2026-07-16
**Status:** Accepted
**Files:** `src/boss/boss_voice_system.gd` (autoload), `src/boss/boss_voice_data.gd`, `src/boss/boss_projectile.gd`+`.tscn` (new), `src/boss/auditor.gd`, `src/boss/distributor.gd`, `src/boss/claim_jumper.gd`, `assets/boss-voices.json`, `scripts/gen_boss_voices.py`

## Context

Bosses were static: a patrol→charge/throw→brief-vulnerable loop with no
ranged threat, no phases, and no personality. Fights read as demos. Goal:
make each boss threatening, phased, and characterful — without breaking the
existing damage-window loop players already understand.

## Decision

### Voice & personality (Part 1)
- `BossVoiceSystem` autoload: `say(source, boss_id, category, force)` picks a
  random line of `category` for `boss_id`, plays it positionally
  (`AudioStreamPlayer2D`), and enforces a 2.2s per-boss cooldown so lines
  never stampede. Ambient taunts auto-fire every 8–12s while a boss is
  `set_active`. `force` bypasses cooldown for story beats (intro/phase/death).
- Text lives in `assets/boss-voices.json` (single source); audio generated by
  `scripts/gen_boss_voices.py` via ElevenLabs into
  `src/assets/sounds/voice/boss/<boss>_<category>_<i>.mp3`. Three distinct
  plan-available voices: Charlie (tax, condescending), Daniel (crystal,
  cold-corporate), Arnold (bandit, gruff). `boss_voice_data.gd` mirrors only
  the per-category COUNTS the runtime needs to pick an index.
- Categories: intro, taunt, mock (player hit), hurt (boss hit), phase50,
  phase25, death — wired into each boss's `_ready`, hitbox-hits-player,
  `take_damage`, phase transition, and `die`.

### AI (Part 2)
- **Aimed projectile** (`boss_projectile.tscn`): flies at the player, damages
  on contact, optional homing. Replaces the old random-direction shard and the
  stationary dynamite drop as the bosses' ranged threat.
- **Three HP-scaled phases** per boss (ratio ≤0.66 → P2, ≤0.33 → P3):
  - **Auditor** — clipboard throws (1 → 2-shot → triple fan), cadence tightens,
    +50% speed in P3, repositioning hops.
  - **Distributor** — aimed ETH orbs (3 → 5 → 5 fast), homing from P2, spread
    widens, cadence tightens.
  - **Claim Jumper** — dynamite now lands ON the player (telegraphed blast
    zone), 1 → 2 → 3 sticks, faster patrol per phase.
- The post-attack VULNERABLE window is preserved as the readable damage tell —
  the new ranged attacks add pressure *between* windows, so the fight stays
  legible but stops being safe.

## Consequences

- Fixed a latent bug: `distributor.gd` shadowed `BossBase.current_phase` (int)
  with its own `Phase` enum var; renamed the state-machine var to
  `current_phase_state` so BossBase's HP-phase tracking works.
- All values remain `@export`/const and data-driven; voice text is JSON.
- Contact damage still only lands during the boss's monitoring windows (bosses
  lack `deal_damage`); ranged projectiles are the always-on threat. Acceptable
  and intentional — a future pass could add always-on contact damage.

## Engine Compatibility

Godot 4.3. Uses `AudioStreamPlayer2D`, `Area2D`, `get_first_node_in_group`,
`Vector2.rotated/lerp`, `SceneTree.create_timer` — all stable.

## GDD Requirements Addressed

CLAUDE.md boss-arena + enemy design; the "commercial release" depth pass.

## Not in this cut (honest scope)

The most elaborate per-phase set-pieces from the brief (Auditor audit-beam
sweep + tax minions; Distributor teleport-pedestals + liquidation beam;
Bandit runaway-cart + mega-bomb countdown) are **not** implemented — they need
bespoke arena geometry and are a follow-up. What shipped is the phased,
ranged, voiced core that makes all three fights threatening today.
